Why this exists

The problem we set out to solve

  • Disconnected tools. Stock, procurement and the lining estimator each lived in separate spreadsheets — numbers never agreed.
  • No accountability. Who issued what, to whom, and who approved it was lost in email and WhatsApp.
  • Stale planning. The estimator showed quantities that didn't reflect what was actually consumed or received.
  • Manual, error-prone. Re-keying SAP codes and quantities invited mistakes that surfaced only at audit.

Accountability in action

Daily issue → End-of-Day commit

The control point of the whole system. Nothing touches the permanent ledger until a HOD signs off.

  • Store Keeper logs issues during the day — each line auto-stamped with who, what, when.
  • Lines sit in a pending state — reviewable, editable, rejectable.
  • HOD reviews, approves or rejects per-line, then commits End-of-Day.
  • On commit, stock is debited and the line becomes an immutable, audited consumption record.
  • Rejected lines are archived (not deleted) — the audit trail is never broken.

The clever part

Strict isolation, live numbers

The estimator's inventory is kept separate from the ERP stock ledger to prevent clutter and corruption — yet its quantities stay current by rolling up real ERP movements at read time.

Available = Initial  +  Received  −  Consumed
  • Initial — the SME-owned baseline, loaded once from the engineering Excel files.
  • Received / Consumed — pulled live from the ERP ledger, never re-keyed.
  • Computed on the fly by a SQL view — nothing is stored, so it can never drift.
  • SME write operations never touch the ERP inventory table, and vice-versa.

Built for integrity

🧪Automated regression harness

525 checks cover schema, every workflow state machine, RBAC locks and the SME integration. Runs before every release; the build is gated on green.

🧾Immutable audit trail

Posted ledger rows are never edited in place. Corrections happen via reversal documents; rejected items are archived, not deleted.

📐Derived, not declared

Live stock and estimator availability are computed from movements — eliminating the "someone typed the wrong number" class of error.

🔒Least-privilege RBAC

Page access is exact-locked per role; multi-site scoping prevents cross-contamination of data between HQ and CNCEC.

Self-healing schema: the database migrates itself forward on startup — new columns, views and seed data apply idempotently with zero manual DB surgery.
