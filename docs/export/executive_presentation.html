1 · The Transformation

Why we upgraded

The legacy tool served us well as a start, but it was built on technology meant for prototypes — and the business had outgrown it.

⛔  Before — Legacy system
  • 🐌Lagging screens — every click reloaded the whole page; busy hours felt frozen.
  • 🔒Database locking — one person saving could block everyone else (single-file database).
  • 👤Weak multi-user support — effectively one heavy user at a time.
  • 📄Manual follow-ups — reports and reminders depended on people remembering.
✅  Now — GI Hub 2.0
  • ⚡Instant response — modern app that updates only what changed on screen.
  • 🗄️True concurrency — an enterprise database designed for many simultaneous writers.
  • 👥Whole teams at once — store keepers, supervisors, HODs and logistics work in parallel.
  • 🤖Automation built-in — approvals, alerts and executive reports happen on their own.
2 · The Technology

What powers it

The world's leading open-source enterprise database — bank-grade data integrity, zero locking issues, decades of proven reliability.

3 · Business Value

SME Smart Calculator 🧮

Ask "what do we need to line these surfaces?" and get the answer in seconds — for one lining system or several at once.

  • 📐Automated material prediction — enter the target surface area(s); the engine computes every component quantity and pack count from the engineering recipes.
  • 🧩Multi-system planning — select several lining systems with one shared or individual per-system areas; requirements aggregate into a single, clean procurement list.
  • 📦Pooled live stock across variant SAP codes — one material can exist under several SAP codes in the ERP; the calculator sums stock across all of them, eliminating the false shortages that used to trigger unnecessary purchases.
  • 🚦Instant shortfall flags — every line shows covered ✓ or short (with the exact quantity), so purchasing decisions are made on facts, not guesses.

4 · Enterprise Security

Defense in depth

Security is layered — an attacker would need to break several independent walls, and staff only ever see what their role allows.

🛡️

Cloudflare Tunnel + Access wall

The server has zero open public ports — it dials out to Cloudflare's network, and every visitor must pass a corporate sign-in wall before even reaching our login page. Encrypted end-to-end.

🎭

Strict role-based access

Store Keeper, Supervisor, HOD, Logistics, Admin — each role sees only its own screens and its own site. Every menu, page and API call is locked to an explicit permission list; SME master data is editable by HOD/Admin only.

🔑

Hardened messaging & OTP

Phone changes require a dual OTP (old number authorizes, new number confirms). OTPs are rate-limited to 3/hour per phone and per source address; inbound WhatsApp webhooks are cryptographically signed, and repeat forgery attempts earn an automatic IP ban.

🤖

Immutable AI boundary

AI features use a separate read-only database identity. Short-lived login tokens rotate automatically, stolen-token reuse revokes all of a user's sessions, and every sensitive action lands in a permanent audit log.
